# 优化项目

## 1. 五层解耦架构（ReAct + MCP）

### 1.1 统一 ReAct 循环
- **旧**: 各节点有独立 LLM 调用逻辑（tool_agent 单次工具调度，knowledge_agent 并行 RAG+搜索）
- **新**: 共享 `_run_react_loop()` 函数，Thought → Action → Observation → Final Answer
- **节点工具隔离**: `tool_agent`（全部工具）、`knowledge_agent`/`lora_agent`（rag_search + web_search）
- **效果**: 消除节点间重复代码，统一 ReAct 格式，工具调用自动重试

### 1.2 MCP 强制解耦
- **旧**: `knowledge_agent`/`lora_agent` 直连 `rag_engine.format_context()`，绕过工具层
- **新**: RAG 注册为 `rag_search` MCP 工具，所有外部数据访问必须通过 MCP 总线
- **效果**: 新增数据源只需注册 MCP 工具，不改 Agent 核心代码；调用日志统一

### 1.3 死代码清理
- 删除 `KNOWLEDGE_AGENT_PROMPT`（被 ReAct 替代）
- 删除 `route_parallel()` + `Send` 导入（定义但未接入图）
- 删除旧 `tool_agent` 单次工具调度逻辑（被共享 ReAct 替代）

## 2. 性能优化

### 2.1 双引擎 RAG 检索加速
- **旧**: 纯向量检索，每次都 BGE embedding
- **新**: 向量(0.6) + BM25(0.4) 混合融合，query 重写，语义分块
- **效果**: 检索精度提升，BM25 零 GPU 开销

### 2.2 LLM 响应缓存
- **旧**: 每次请求调用 DeepSeek API（8~30s）
- **新**: SHA256(system_prompt + user_message + temperature) 缓存，LRU 1000 条，TTL 1h
- **效果**: 缓存命中 → **5ms**（218 QPS），跳过 BGE + 搜索 + LLM 全部流程

### 2.3 异步改造
- **RAG**: `sync` → `async def`，修复 `Cannot run the event loop while another loop is running`
- **历史记录**: `aioredis` + TCP 探针，2ms 检测 Redis 不可用

## 3. 稳定性优化

### 3.1 Circuit Breaker（断路器）
- 每把 API Key **三态**: CLOSED → OPEN → HALF_OPEN
- 连续失败 → 熔断 10s，指数退避至 120s
- 半开恢复检测，3 把 Key 轮询

### 3.2 节点级重试
- `_call_llm` 内部遍历全部可用 Key 重试
- 全部熔断时降级返回中文提示

### 3.3 逐节点超时
| 节点 | 超时 |
|------|------|
| intent | 10s |
| code_agent / knowledge_agent / data_agent | 60s |
| chat_agent / tool_agent / lora_agent | 30s |
| supervisor | 10s |
| merger | 15s |

### 3.4 Supervisor 重分类
- 如果回答以 `不知道/我不清楚/无法回答` 开头 → supervisor 重新分类 → 改走不同 worker
- 避免同一个错误节点重复失败

### 3.5 Redis 降级
- `redis-py 8.0.0` `socket_connect_timeout` 参数无效，25s 超时
- 修复: `asyncio.open_connection(host, port, timeout=2)` TCP 探针 → Redis 不可用直接跳过 < 2ms

## 4. 成本优化

### 4.1 LoRA 免费推理层
- 体育/世界杯关键词 → `lora_agent` → Qwen2.5-3B（本地 GPU）
- 一次训练（34 Q&A，10 epochs，96s）永久免费推理
- 质量门禁: 答案 < 10 字符或命中 stale 模式 → 回退 ReAct-MCP

### 4.2 关键词范围收紧
- 仅足球相关关键词匹配 LoRA，避免其他体育项目路由到未训练的适配器

---

## 5. 架构进化

### 5.1 自实现 FSM → LangGraph
- 原: `if/elif` 状态机，硬编码路由
- 现: `StateGraph(AgentState)` + `TypedDict` + `Send()` 并行 + `astream` 流式

### 5.2 单次工具调度 → ReAct 循环
- 原: tool_agent 单次 LLM 决策调用一个工具
- 现: 共享 `_run_react_loop()`，Thought→Action→Observation→Final Answer
- knowledge_agent / tool_agent / lora_agent 降级统一走此路径

### 5.3 直连 RAG → MCP 工具总线
- 原: knowledge_agent 直接调 rag_engine.format_context()
- 现: 通过 MCP `rag_search` 工具间接调用

### 5.4 响应流式化
- `run_agent_stream`: `graph.astream()` 事件发射
- LoRA 快速路径旁路图（保持低延迟）

### 5.5 完整节点拓扑
```
intent ──┬── code_agent ────── supervisor ──┬── code_agent
         ├── knowledge_agent ─supervisor────┤── knowledge_agent
         ├── data_agent ───────supervisor───┤── data_agent
         ├── tool_agent ────────────────────┤── merger → END
         ├── chat_agent ────────────────────└── END
         └── lora_agent ────────────────────── END
```
