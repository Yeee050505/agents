# 项目需求文档

## 一句话

搭建一个基于 **五层解耦架构（L1 LLM + L2 RAG + L3 MCP + L4 ReAct + L5 React）** 的多智能体协作平台，支持私有知识库、联网搜索、LoRA 本地推理、流式输出、高可用。

---

## 用户需求

### 1. 智能对话
用户直接提问，系统自动选择最合适的模型和策略回答，无需用户选择模式。

- 闲聊 → chat agent
- 编程问题 → code agent
- 知识问答 → knowledge agent（ReAct → MCP → RAG + 搜索）
- 数据分析 → data agent
- 工具调用 → tool agent（ReAct → MCP → 全部工具）
- 体育/世界杯 → lora agent（L1 LoRA 本地推理）
- 全部自动意图识别路由，无需手动切换

### 2. 私有知识库（L2 RAG）
用户上传文档，系统基于文档内容回答，不依赖大模型通用知识。

- 支持 .txt / .md / .pdf
- 上传后自动语义分块、BGE 嵌入、向量+BM25 混合索引
- 回答时通过 L3 MCP 的 `rag_search` 工具检索，节点禁止直连

### 3. 实时信息联网搜索
大模型知识有截止日期，实时数据自动联网补充。

- 通过 L3 MCP 的 `web_search` 工具（Bing + 百度并发）
- ReAct 循环内自主决定何时搜索、搜什么、搜几次
- 答案质量检测到"知识过时"标记时自动重搜重答

### 4. 多轮对话上下文
保留最近 20 轮对话，1 小时无活动自动清除。

- 基于 session_id 隔离会话
- Redis 存储（TTL=3600s），不可用时降级内存
- L4 ReAct 循环通过 `_build_context()` 将历史注入 LLM

### 5. LoRA 本地推理（L1）
特定领域问题走本地 GPU 推理，降低 API 成本。

- 体育/世界杯关键词自动路由到 `lora_agent`
- Qwen2.5-3B + LoRA 适配器，RTX 4060 8GB 运行
- 质量门禁：低质量回答自动降级到 ReAct-MCP 知识路径

### 6. 高可用
API key 不能成为单点故障。

- 3 个 DeepSeek key 池化，轮询负载
- 单个 key 失败自动熔断（CLOSED→OPEN→HALF_OPEN），指数退避 10s→120s
- `_call_llm` 主实例失败后遍历剩余健康实例
- 全部熔断返回友好降级文案

### 7. 服务稳定
系统不能被单一用户拖垮。

- 全局限流 100 请求/秒
- 用户级限流 10 请求/秒
- 内存令牌桶，零依赖

### 8. 快速响应
用户不想等 30 秒才看到回复。

- SSI 流式输出：1s 首 token，逐字渲染
- LLM 响应缓存：重复问题 5ms 返回
- ReAct 循环超时兜底：逐节点 10s~60s

### 9. 五层架构解耦
所有外部数据访问必须经过 MCP，节点禁止直连 RAG/搜索。

- L1：LLM 推理引擎（DeepSeek API + Qwen LoRA）
- L2：知识数据底座（BM25+向量混合检索）
- L3：MCP 工具总线（注册中心、统一调用、调用日志）
- L4：ReAct 决策循环（Thought→Action→Observation→Final Answer）
- L5：React SPA（对话交互 + 知识库管理 + 状态监控）

### 10. 账号安全
- 注册 / 登录 / JWT 鉴权
- 密码 SHA256 加密
- 访客模式无需登录
